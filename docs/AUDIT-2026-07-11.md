# Honesty Audit — 2026-07-11

Adversarial self-audit of everything previously claimed "implemented" (PD-5).
Method: (1) mechanically verify every test name claimed in a checked
acceptance criterion exists; (2) CONV-21 dry run — for every requirement ID in
every spec, does an ID-named test exist; (3) adversarial read of the decision
paths for numbers that could be rosier than reality. Findings were **fixed in
the same change**, each with a `regression_audit*` test.

## Bugs found (all fixed, all with regression tests)

| # | Severity | Finding | Fix |
|---|---|---|---|
| 1 | **High** | `Metrics` claimed "after costs" but per-trade P&L fed to expectancy was **gross of fees**, and the SIM-8 "2×-cost" stress therefore effectively priced only 1× — inflating the exact numbers gate G1 trusts. | `Accountant::apply_fill` now returns a `FillOutcome` attributing each closing fill's fee **plus the pro-rata entry fees it releases**; the engine records net P&L. The accounting identity (which needs gross realized + separate fee term) is unchanged. `regression_audit1_expectancy_is_net_of_entry_and_exit_fees` pins the arithmetic (gross 5.00 → net 2.85 → 2×-stress 0.70). |
| 2 | Medium | The trade-print rule filled resting limits on a print exactly **AT** the limit price. SIM-2 is explicit: fills require trading **through** — touching is not filling. At-price fills are optimistic. | Strict inequalities. `regression_audit2_trade_print_at_price_does_not_fill`. |
| 3 | Medium | All dead-man alerts share the id `process-deadman` and the router deduped per id — **process B dying inside process A's dedupe window was silently suppressed**. | `Alert.dedupe_key` (defaults to id; dead-man sets `process-deadman/{proc}`). `regression_audit3_deadman_alerts_not_cross_deduped`. |
| 4 | **High** | The sim's docs and SIM-5 claimed it "runs the PRODUCTION RiskGate" — `mp_risk::evaluate` was **never called**; sized intents went straight to the fill model. | The gate now sits between sizing and fills: RG-2 allowlist from the feed, RG-3/4/5 notionals, RG-6 price sanity, RG-7 event-time rate limit, RG-8/9 daily-loss vs day-start equity, RG-10 kill switches (tripable in sim), verdicts recorded in the decision-log hash. `sim_5_production_risk_gate_gates_every_sized_intent` proves blocking limits ⇒ zero fills and RG-10 ⇒ zero fills. |

## Traceability repairs (CONV-21 / W-2)

- `ops4_…`/`res6_…`-style test names didn't match the `{prefix}_{n}_` needle
  the guardrail greps for — renamed (`ops_4_…`, `res_6_…`, `test_res_1_…`).
- The guardrail's Python arm grepped `def res_5…`, which can never match a
  collectible pytest test (pytest mandates the `test_` prefix) — the check was
  structurally blind to all Python tests. It now accepts `def test_{id}_…`.
  This is a *correction of a broken check*, not a loosening: previously it
  could only ever fail, and only for specs marked `implemented`.
- Added genuinely-covering ID-named tests where the behavior already existed
  but no test traced to the ID: `sim_3` (latency deferral), `sto_4` (dataset
  global-order read), `rsk_8` (SizingTrace recomposes the formula),
  `ops_1/5/8/10` (deployment artifacts pinned), `res_8` (prompt-version
  headers). Plus new real functionality: FEA-5 NaN suppression counters,
  FEA-9 offline-feature live refusal, OPS-7 disk/clock/keyfile checkers,
  RSK-5 `dd_budget_from_mc = p95(maxDD)×1.25`.

## Verdict

The claim "all specs are implemented" **does not hold**, and the audit found
four real defects in what was claimed — two of which (fees in expectancy, the
missing gate) would have made bad strategies look better than they are, which
is precisely the failure mode this system exists to prevent. All four are
fixed and pinned by regression tests. Statuses: 001 `implemented`; 000,
002–010 `implementing` with their remainders named above; 011 `draft` (Phase-7
by recorded decision).

Gates at time of audit: 37 Rust test binaries + 24 Python tests passing;
clippy clean (default, `live-ws`, `live-http`); fmt clean; guardrails green
(including the new CONV-3 and corrected CONV-21 checks).
